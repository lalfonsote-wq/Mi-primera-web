Clase 1

Cómo convertir un sitemap en un menú horizontal responsive

En esta clase vamos a construir la base de un sitio del Instituto Tecnológico Unión Panamericana usando: container, menú de primer nivel, Flexbox, media query y una estructura reutilizable.

Objetivo visual:

  • En PC el menú debe verse horizontal.
  • En móvil el menú debe verse vertical.
  • El contenido debe mantenerse centrado y limpio.
  • Cada opción principal debe enlazar a una subpágina.

Esta presentación incluye explicación y código listo para copiar.

Clase 2

Estructura del proyecto

Para este ejemplo usaremos una estructura sencilla de archivos HTML separados y un solo CSS compartido. 

proyecto/
│
├── index.html
├── nosotros.html
├── admisiones.html
├── calendario.html
├── contacto.html
└── styles.css

La idea es que el header, el container y el menú se repitan en todas las páginas para mantener consistencia.

Ventaja

El menú siempre se ve igual y el usuario puede navegar sin perder orientación.

Lógica

Cada enlace del menú principal lleva a una página distinta, como nosotros.html o contacto.html.

Clase 3

HTML base con container y menú principal

Este es el HTML principal para crear el container y el menú de primer nivel. 

<header class="header">
  <div class="container">
    <h1 class="logo">Instituto Tecnológico Unión Panamericana</h1>

    <nav>
      <ul class="menu">
        <li><a href="index.html">Inicio</a></li>
        <li><a href="nosotros.html">Nosotros</a></li>
        <li><a href="admisiones.html">Admisiones</a></li>
        <li><a href="calendario.html">Calendario</a></li>
        <li><a href="contacto.html">Contacto</a></li>
      </ul>
    </nav>
  </div>
</header>

Aquí ul.menu contiene el primer nivel del menú y cada li representa una opción principal.

Clase 4

¿Para qué sirve el container?

.container {
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;
}

width: 90%

Hace que el contenido use casi todo el ancho disponible, pero dejando respiración en los lados.

max-width: 1200px

Evita que en pantallas grandes el contenido se haga demasiado ancho.

margin: 0 auto

Centra horizontalmente el bloque dentro de la pantalla.

Responsive

El container se adapta bien tanto en móvil como en desktop.

Clase 5

Menú horizontal con Flexbox

En PC usamos Flexbox para colocar el menú en una sola fila. 

.menu {
  list-style: none;
  display: flex;
  justify-content: center;
  gap: 20px;
  flex-wrap: wrap;
}

.menu li a {
  display: block;
  text-decoration: none;
  background-color: #c8895d;
  color: white;
  padding: 12px 18px;
  border-radius: 8px;
}
  • Inicio
  • Nosotros
  • Admisiones
  • Calendario
  • Contacto

display: flex pone los elementos uno al lado del otro. justify-content: center los centra. gap separa los botones. flex-wrap: wrap evita que se rompan si no caben.

Clase 6

Media query para móvil

Cuando la pantalla es pequeña, el menú debe pasar de horizontal a vertical. 

@media (max-width: 768px) {
  .menu {
    flex-direction: column;
    align-items: stretch;
    gap: 10px;
  }

  .menu li a {
    width: 100%;
    text-align: center;
  }
}

flex-direction: column

Coloca las opciones una debajo de otra.

align-items: stretch

Hace que los elementos ocupen el ancho disponible.

PC = horizontal / Móvil = vertical

Clase 8

Duplicar la estructura para las demás páginas

Para reutilizar el mismo menú, copias el mismo bloque de header en cada archivo HTML. 

<!-- nosotros.html -->
<header class="header">
  <div class="container">
    <h1 class="logo">Instituto Tecnológico Unión Panamericana</h1>
    <nav>
      <ul class="menu">
        <li><a href="index.html">Inicio</a></li>
        <li><a href="nosotros.html">Nosotros</a></li>
        <li><a href="admisiones.html">Admisiones</a></li>
        <li><a href="calendario.html">Calendario</a></li>
        <li><a href="contacto.html">Contacto</a></li>
      </ul>
    </nav>
  </div>
</header>

Eso permite que la navegación sea consistente en todo el sitio.

Clase 9

Cómo enlazar a subpáginas internas

Cada sección principal puede tener una página interna con enlaces a sus temas específicos. 

<main class="container">
  <h2>Nosotros</h2>
  <ul class="subpages">
    <li><a href="historia.html">Historia</a></li>
    <li><a href="mision.html">Misión</a></li>
    <li><a href="valores.html">Valores</a></li>
    <li><a href="docentes.html">Docentes</a></li>
    <li><a href="coordinadores.html">Coordinadores</a></li>
  </ul>
</main>
.subpages {
  margin-top: 20px;
  list-style: none;
}

.subpages li {
  margin-bottom: 10px;
}

.subpages li a {
  text-decoration: none;
  color: #8b5e3c;
  font-weight: bold;
}

Clase 11

Por qué este diseño es mobile friendly

  • Usa meta viewport para adaptarse a la pantalla del celular.
  • El container limita el ancho y evita desbordes.
  • Flexbox permite reorganizar el menú fácilmente.
  • La media query cambia el menú a vertical en pantallas pequeñas.
  • Los enlaces tienen suficiente padding para tocarse con el dedo.
  • La navegación de esta presentación está fija abajo para fácil acceso.

Buena práctica

Aunque esta presentación usa JavaScript para pasar slides, el menú del sitio del instituto puede funcionar solo con HTML y CSS tal como lo viste en los ejemplos.
